# 宏观驱动ETF配置策略 — 项目架构设计（初始版）

## 项目目标与原则

### 核心目标
- 为宏观驱动ETF配置策略构建清晰、模块化的项目结构
- 建立标准化的文件组织体系，确保后续开发的可扩展性和可维护性
- 定义明确的模块职责边界，为增量开发奠定基础

### 当前阶段定位
- **仅创建目录与空文件**，暂不实现具体业务逻辑
- **暂不配置LLM详细参数**，预留智能体集成接口
- **专注架构设计**，确保结构合理性和可扩展性

### 设计原则
- **结构扁平**：减少层级嵌套，提高代码可读性
- **职责单一**：每个模块仅负责一项核心功能
- **无冗余抽象**：避免过度设计，保持实现简洁直接
- **接口清晰**：模块间交互方式明确，降低集成复杂度

## 项目目录结构

```text
Q-Macro/
├── pyproject.toml                 # 项目配置文件（基础元信息）
├── README.md                      # 项目说明文档
├── .gitignore                     # Git忽略规则配置
│
├── data/                          # 本地数据目录（建议不纳入版本控制）
│   ├── policy_texts/              # 政策原文存储（支持.txt或.pdf格式）
│   ├── etf_prices/                # ETF量价数据（按ticker命名的CSV文件）
│   └── etf_meta.csv               # ETF元数据（包含ticker, name, index_name字段）
│
├── src/                           # 核心源码目录
│   ├── core/                      # 核心策略模块（纯Python函数实现）
│   │   ├── __init__.py            # 模块初始化文件
│   │   ├── macro_regime.py        # 宏观状态与景气度分析模块
│   │   ├── signal_fusion.py       # 政策与宏观信号融合模块
│   │   ├── market_calibration.py  # 基于量价的市场校准模块
│   │   └── portfolio_builder.py   # ETF组合构建与调仓模块
│   │
│   └── agents/                    # LLM智能体模块（后续实现）
│       ├── __init__.py            # 模块初始化文件
│       ├── policy_interpreter.py  # 政策文本解析智能体
│       ├── theme_mapper.py        # ETF主题标签映射智能体
│       └── report_writer.py       # 投资报告生成智能体
│
└── scripts/                       # 可执行脚本目录
    ├── fetch_macro_data.py        # 宏观经济数据获取脚本（基于AKShare）
    └── run_monthly_pipeline.py    # 月度策略执行主流程入口
```

## 文件内容规范（初始状态）

### 配置文件规范

**pyproject.toml**
```toml
[project]
name = "Q-Macro"
version = "0.1.0"
description = "Macro-driven ETF allocation strategy for Chinese market"
```

### 源码文件规范

**src/core/ 与 src/agents/ 目录下的Python文件**
- **初始状态**：仅包含模块文档字符串（`"""Module docstring."""`）
- **依赖管理**：暂不导入任何第三方依赖
- **实现风格**：后续以函数式编程为主，避免过度使用类

**scripts/ 目录下的Python文件**
- **初始状态**：创建空文件结构
- **功能定位**：作为模块调用入口，负责组合各核心模块的功能

### 数据目录规范

**data/ 目录**
- **初始化**：仅创建目录结构，不包含示例文件
- **数据要求**：用户需自行准备以下数据文件：
  - `etf_meta.csv`：ETF基本信息，至少包含 `ticker`, `name`, `index_name` 三个字段
  - 政策文本文件：如 `2026_q1_politburo.txt`，存储于 `policy_texts/` 目录
  - ETF价格文件：如 `510300.csv`，包含 `date, close, volume, iopv` 等字段，存储于 `etf_prices/` 目录

## 开发流程建议

### 阶段一：核心策略模块开发
- 优先实现 `src/core/` 目录下的模块
- 采用纯规则实现，结合AKShare获取宏观数据
- 确保核心逻辑独立于LLM，提高系统稳定性

### 阶段二：数据获取与处理
- 实现 `scripts/fetch_macro_data.py`
- 验证AKShare接口可用性，确保数据获取流程顺畅
- 建立数据预处理与标准化机制

### 阶段三：智能体集成
- 实现 `src/agents/` 目录下的智能体模块
- 集成LLM能力，实现政策解读、ETF标签映射和报告生成功能
- 确保智能体与核心模块的无缝对接
