# 宏观状态与景气度模块开发方案

## 模块概述

**模块1：宏观状态与景气度**（`src/core/macro_regime.py`）是 Q-Macro 策略系统的"宏观感知中枢"，负责基于宏观经济指标识别当前经济周期状态并生成景气度评分，为后续资产配置决策提供核心依据。

---

## 模块目标

给定目标日期（如 `2025-12-31`），从 `data/processed_macro_data/` 目录读取最新可用的宏观经济指标数据，输出结构化的宏观状态与景气度评分结果。

### 设计原则
- **纯硬编码实现**：无 LLM 依赖，确保逻辑可复现
- **标准化接口**：输入仅为日期字符串，输出为标准字典结构
- **可测试性**：所有逻辑支持单元测试和历史回测
- **鲁棒性**：具备错误处理和数据缺失时的容错机制

---

## 输入数据说明

| 文件名 | 关键列 | 频率 | 用途 |
|--------|-------|------|------|
| `pmi.csv` | `date`, `value` | 月度 | 经济增长领先指标 |
| `cpi.csv` | `date`, `yoy` | 月度 | 消费端通胀指标 |
| `ppi.csv` | `date`, `yoy` | 月度 | 生产端通胀指标 |
| `industrial_production.csv` | `date`, `yoy` | 月度 | 工业增长指标 |
| `gdp.csv` | `date`, `yoy` | 季度 | 经济增长辅助验证（可选） |

### 数据处理说明
- **列名容错**：支持常见时间列名（如 `时间`、`date`、`统计时间` 等）
- **日期对齐**：统一处理为 `YYYY-MM` 格式，确保跨指标时间一致性
- **数据完整性**：若最新数据缺失，支持回退到历史最新值

## 核心逻辑实现

### 1. 数据加载与对齐
- 读取各宏观指标 CSV 文件
- 标准化时间列名为 `date`
- 将日期格式统一转换为 `YYYY-MM` 格式
- 按日期降序排序，确保最新数据在前
- 基于目标日期确定使用的最新数据月份

### 2. 指标值提取
- 从排序后的数据中提取最新月份的指标值
- 处理数据缺失情况：若最新月份数据缺失，回退到上一月数据
- 记录实际使用的数据月份，确保可追溯性

### 3. 四象限经济状态判断

```python
growth = (pmi > 50.0) or (industrial_yoy > 5.0)
inflation = (cpi_yoy > 2.5) or (ppi_yoy > 0.0)

if growth and inflation:
    regime = "overheat"  # 过热
elif growth and not inflation:
    regime = "recovery"  # 复苏
elif not growth and inflation:
    regime = "stagflation"  # 滞胀
else:
    regime = "recession"  # 衰退
```

### 4. 动量指标计算
- 从各指标序列中提取最近3个非空值
- 计算变化率：采用线性斜率或简单差值方法
- 标准化处理：将变化率缩放到合理范围（如除以10）

### 5. 权益友好度评分计算

```python
# 基础分：PMI 映射到 [0,1] 区间
base_score = max(0.0, min(1.0, (pmi - 45.0) / 10.0))

# 动量加分：基于经济增长动量
momentum_bonus = 0.1 * growth_momentum  # 假设 momentum ∈ [-0.5, 0.5]

# 通缩惩罚：PPI长期为负时的调整
deflation_penalty = 0.1 if ppi_yoy < -2.0 else 0.0

# 计算最终评分并约束在 [0,1] 区间
score = base_score + momentum_bonus - deflation_penalty
score = max(0.0, min(1.0, score))
```
